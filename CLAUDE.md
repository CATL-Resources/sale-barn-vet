# Working notes for this project

## How to write and explain things (keep to this)
Use plain, everyday language. Drop the jargon and heavy terms.
- Say what a thing *does*, not what it's *called*.
- For every action item, spell out the real work in concrete words — what to
  change, where, and what it should do — so anyone can act on it without having
  to decode terminology first.
- Short sentences. If a simpler word works, use it.

This applies everywhere: chat replies, plans, pull request descriptions, commit
messages, and docs.
